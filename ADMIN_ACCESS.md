# 🛠️ Руководство по админ панели SharikPlus

Полное руководство по использованию административной панели интернет-магазина воздушных шаров.

## 🌐 Доступ к админ панели

### 🔗 Основной интерфейс
**URL**: http://localhost:3001/api/docs

Админ панель реализована через **Swagger UI** - современный интерфейс для управления API, который предоставляет все необходимые административные функции.

### 🔑 Авторизация
- **Admin**: admin@sharikplus.ru / admin123
- **Manager**: manager@sharikplus.ru / manager123

## 📊 Возможности админ панели

### 1. 📦 Управление товарами

#### Просмотр всех товаров
- **Endpoint**: `GET /products`
- **Описание**: Получить список всех товаров с пагинацией
- **Параметры**: 
  - `skip` - количество пропускаемых записей
  - `take` - количество записей для получения
  - `search` - поиск по названию

#### Создание нового товара
- **Endpoint**: `POST /products`
- **Описание**: Добавить новый товар в каталог
- **Данные**:
  ```json
  {
    "name": "Набор шаров 'Праздник'",
    "slug": "balloon-party-set",
    "description": "Яркий набор воздушных шаров для праздника",
    "price": 1500,
    "sku": "BALLOON-PARTY-001",
    "active": true,
    "featured": true
  }
  ```

#### Обновление товара
- **Endpoint**: `PUT /products/{id}`
- **Описание**: Изменить существующий товар
- **Параметры**: ID товара в URL

#### Удаление товара
- **Endpoint**: `DELETE /products/{id}`
- **Описание**: Удалить товар из каталога
- **Параметры**: ID товара в URL

#### Просмотр конкретного товара
- **Endpoint**: `GET /products/{id}`
- **Описание**: Получить детальную информацию о товаре

### 2. 📋 Управление заказами

#### Просмотр всех заказов
- **Endpoint**: `GET /orders`
- **Описание**: Получить список всех заказов
- **Фильтры**:
  - По статусу (PLACED, CONFIRMED, FULFILLED, CANCELLED)
  - По дате создания
  - По клиенту

#### Создание заказа
- **Endpoint**: `POST /orders`
- **Описание**: Создать новый заказ
- **Данные**:
  ```json
  {
    "email": "customer@example.com",
    "phone": "+7 (999) 123-45-67",
    "items": [
      {
        "productId": "product-id",
        "quantity": 2,
        "price": 1500
      }
    ],
    "shipping": {
      "address": "Москва, ул. Примерная, д. 1",
      "method": "courier"
    }
  }
  ```

#### Изменение статуса заказа
- **Endpoint**: `PUT /orders/{id}/status`
- **Описание**: Обновить статус заказа
- **Статусы**:
  - `PLACED` - Размещён
  - `CONFIRMED` - Подтверждён
  - `FULFILLED` - Выполнен
  - `CANCELLED` - Отменён

#### Детали заказа
- **Endpoint**: `GET /orders/{id}`
- **Описание**: Получить полную информацию о заказе

### 3. 👥 Управление пользователями

#### Просмотр пользователей
- **Endpoint**: `GET /users`
- **Описание**: Получить список всех пользователей
- **Фильтры**:
  - По роли (USER, MANAGER, ADMIN)
  - По статусу активности
  - По дате регистрации

#### Регистрация пользователя
- **Endpoint**: `POST /auth/register`
- **Описание**: Создать нового пользователя
- **Данные**:
  ```json
  {
    "email": "newuser@example.com",
    "password": "securepassword",
    "name": "Имя Пользователя",
    "role": "USER"
  }
  ```

#### Авторизация
- **Endpoint**: `POST /auth/login`
- **Описание**: Войти в систему
- **Данные**:
  ```json
  {
    "email": "admin@sharikplus.ru",
    "password": "admin123"
  }
  ```

#### Изменение роли пользователя
- **Endpoint**: `PUT /users/{id}/role`
- **Описание**: Изменить роль пользователя
- **Роли**:
  - `USER` - Обычный пользователь
  - `MANAGER` - Менеджер
  - `ADMIN` - Администратор

### 4. 📊 Статистика и отчёты

#### Общая статистика
- **Endpoint**: `GET /admin/stats`
- **Описание**: Получить общую статистику магазина
- **Данные**:
  - Количество товаров
  - Количество заказов
  - Количество пользователей
  - Общая выручка

#### Статус системы
- **Endpoint**: `GET /health`
- **Описание**: Проверить работоспособность системы
- **Ответ**: `{"status": "ok", "timestamp": "..."}`

## 🔧 Как использовать Swagger UI

### Шаг 1: Открытие админ панели
1. Откройте браузер
2. Перейдите по адресу: http://localhost:3001/api/docs
3. Вы увидите интерфейс Swagger UI с полным списком API endpoints

### Шаг 2: Авторизация (если требуется)
1. Найдите раздел "Auth" в Swagger UI
2. Откройте endpoint `POST /auth/login`
3. Нажмите "Try it out"
4. Введите данные администратора:
   ```json
   {
     "email": "admin@sharikplus.ru",
     "password": "admin123"
   }
   ```
5. Нажмите "Execute"
6. Скопируйте полученный JWT токен
7. Нажмите кнопку "Authorize" в верхней части страницы
8. Введите токен в формате: `Bearer YOUR_TOKEN`

### Шаг 3: Выполнение операций
1. Выберите нужный endpoint (например, `GET /products`)
2. Нажмите "Try it out"
3. Заполните параметры (если требуется)
4. Нажмите "Execute"
5. Просмотрите результат в разделе "Response"

### Шаг 4: Создание/Обновление данных
1. Выберите POST или PUT endpoint
2. Нажмите "Try it out"
3. Заполните JSON в поле "Request body"
4. Нажмите "Execute"
5. Проверьте результат

- **Новые пользователи**: Анализируйте рост аудитории
- **Выручка**: Отслеживайте финансовые показатели

1. **Ежедневно**:

2. **Еженедельно**:
   - Добавление новых товаров
   - Анализ популярных товаров
   - Обновление цен

3. **Ежемесячно**:

   - Оптимизация каталога

## 🔒 Безопасность

### Рекомендации по безопасности
1. **Регулярно меняйте пароли** администраторов
2. **Используйте сложные пароли** (минимум 8 символов)
3. **Ограничивайте доступ** только необходимым сотрудникам
4. **Регулярно проверяйте логи** на подозрительную активность
5. **Делайте резервные копии** базы данных

### Роли и права доступа
- **ADMIN**: Полный доступ ко всем функциям
- **MANAGER**: Управление товарами и заказами
- **USER**: Только просмотр собственных заказов

## 🆘 Поддержка

### Частые проблемы
1. **Не могу войти в админ панель**: Проверьте правильность email и пароля
2. **Ошибка 401**: Необходима авторизация через JWT токен
3. **Ошибка 403**: Недостаточно прав доступа
4. **Ошибка 500**: Проблема на сервере, проверьте логи
