# Информация о проекте SharikPlus

## 📋 Описание

SharikPlus - это полностью готовый к развёртыванию интернет-магазин для продажи воздушных шаров, разработанный с использованием современных технологий и лучших практик разработки веб-приложений в 2025 году.

## �� Основные цели проекта

1. **Создание полнофункционального интернет-магазина** без интеграции платёжных систем
2. **Генерация PDF-документов** для оформления заказов
3. **Модульная и масштабируемая архитектура** для будущего развития
4. **Современный UI/UX** с поддержкой тёмной/светлой темы
5. **Полная SEO-оптимизация** и PWA функциональность

## 🏗️ Архитектура

### Монорепозиторий
- **apps/frontend** - Next.js 14+ приложение
- **apps/backend** - NestJS 11+ API
- **packages/ui** - Общие UI компоненты
- **packages/config** - Конфигурации и пресеты

### Backend (NestJS)
- **Модульная архитектура** с разделением ответственности
- **Prisma ORM** для работы с PostgreSQL
- **Redis** для кэширования и сессий
- **JWT аутентификация** с refresh токенами
- **GraphQL API** + REST endpoints
- **Swagger документация** для API

### Frontend (Next.js)
- **App Router** для современной маршрутизации
- **React 19** с новейшими возможностями
- **TypeScript** для типобезопасности
- **Tailwind CSS 4** для стилизации
- **Framer Motion** для анимаций
- **PWA** с офлайн-режимом

## 📊 База данных

### Основные сущности
- **Users** - пользователи системы
- **Products** - товары магазина
- **Categories** - категории товаров
- **Orders** - заказы пользователей
- **Cart** - корзина покупок
- **Assets** - файлы и изображения

### Особенности
- **PostgreSQL 16+** как основная БД
- **Redis** для кэширования и сессий
- **Prisma** для type-safe работы с БД
- **Миграции** для управления схемой
- **Сиды** для тестовых данных

## 🌐 API Endpoints

### Публичные
- `GET /api/products` - список товаров
- `GET /api/products/:slug` - товар по slug
- `POST /api/cart/add` - добавление в корзину
- `POST /api/orders/checkout` - оформление заказа

### Защищённые
- `GET /api/users/profile` - профиль пользователя
- `GET /api/orders` - заказы пользователя
- `PUT /api/users/profile` - обновление профиля

### Административные
- `GET /api/admin/dashboard` - статистика
- `GET /api/admin/orders` - управление заказами
- `GET /api/admin/users` - управление пользователями
